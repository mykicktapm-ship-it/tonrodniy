Provably Fair Protocol v1.2

lock

1. The Setup

Before the round begins, the server generates a random seed and gives you the hash (SHA256). This proves we committed to the result before you even placed a bet.

edit_square

2. Your Influence

You provide a Client Seed. This is combined with the Server Seed. Since the server doesn't know your seed in advance, it cannot rig the outcome.

check_circle

3. Verification

Once the round ends, the server reveals the unhashed seed. You can verify that the hash matches and the result was calculated correctly.

Reference Implementation (Javascript)

const crypto = require('crypto');

function getResult(serverSeed, clientSeed, nonce) {
  // 1. Create HMAC_SHA256
  const hmac = crypto.createHmac('sha256', serverSeed);
  
  // 2. Update with Client Seed + Nonce
  hmac.update(`${clientSeed}:${nonce}`);
  
  // 3. Get Hex String
  const hex = hmac.digest('hex');
  
  // 4. Take first 8 chars (32 bits)
  const sub = hex.substring(0, 8);
  
  // 5. Convert to decimal
  const decimal = parseInt(sub, 16);
  
  // 6. Modulo available slots (e.g., 15)
  return decimal % 15;
}
